Community

Deckhouse is an Open Source project, and everyone is warmly welcome to contribute to it on GitHub.

To make Deckhouse development vigorous and sustainable, its platform is offered in two editions:

Community Edition (CE)

An Open Source and free of charge version providing key functionality of Deckhouse. Its code is available under the Apache 2.0 license.

Enterprise Edition (EE)

A commercial version that boasts additional features and various subscription options. Its code is also open, but it’s not free, and the license is not Open Source.
The development process of both editions is public.

Technical core

Deckhouse Platform follows the upstream version of Kubernetes, building all its features and configurations on top of it.

The basic functionality of Deckhouse Platform is implemented via two building blocks: shell-operator & addon-operator. Both of them have been available as Open Source projects since April/May 2019.

Aside from Deckhouse, various vendors (including KubeSphere, Confluent, and Adobe, just to name a few) and DevOps/Kubernetes enthusiasts have adopted these tools to implement their own instruments.

To sum it up, in a nutshell, Deckhouse Platform — as a Kubernetes distribution — is an upstream Kubernetes packed with a bunch of modules based on addon-operator.

Shell-operator

A tool to create Kubernetes operators (any logic to be executed when some event in K8s is triggered).

Addon-operator

A system to manage Kubernetes modules based on shell-operator in a consistent, automated manner.
